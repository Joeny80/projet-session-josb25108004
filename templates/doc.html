Documentation API

Référence des services REST du projet Contraventions Alimentaires.

Accueil Faire une plainte
GET /contrevenants?du=YYYY-MM-DD&au=YYYY-MM-DD

Retourne la liste des contraventions émises entre deux dates spécifiées.

Paramètres
Nom Type Requis Description
du string Oui Date de début au format YYYY-MM-DD
au string Oui Date de fin au format YYYY-MM-DD
Exemple
GET /contrevenants?du=2022-05-08&au=2024-05-15
GET /api/restaurants

Retourne la liste distincte des établissements utilisés dans la recherche A6.

Exemple de réponse
[
  { "con_etablissement": "Restaurant ABC" },
  { "con_etablissement": "Bistro XYZ" }
]
GET /api/contraventions?restaurant=Nom

Retourne toutes les infractions d’un restaurant spécifique.

Paramètres
Nom Type Requis Description
restaurant string Oui Nom de l’établissement ou partie du nom
Exemple
GET /api/contraventions?restaurant=Restaurant%20ABC
GET /api/etablissements

Retourne les établissements triés par nombre d’infractions décroissant, en JSON.

Exemple de réponse
[
  { "con_etablissement": "Restaurant Problématique", "nombre_infraction": 15 },
  { "con_etablissement": "Bistro XYZ", "nombre_infraction": 8 }
]
GET /api/etablissements.xml

Retourne les mêmes données que C1 au format XML UTF-8.

Exemple de réponse
<etablissements>
  <item>
    <con_etablissement>Restaurant Problématique</con_etablissement>
    <nombre_infraction>15</nombre_infraction>
  </item>
</etablissements>
GET /api/etablissements.csv

Retourne les mêmes données que C1 au format CSV UTF-8.

Exemple de réponse
con_etablissement,nombre_infraction
Restaurant Problématique,15
Bistro XYZ,8
POST /api/inspection

Crée une nouvelle demande d’inspection. Le corps JSON est validé par schéma.

Corps JSON attendu
Champ Type Requis Description
nom_etablissement string Oui Nom de l’établissement
adresse string Oui Adresse de l’établissement
ville string Oui Ville
date_visite string Oui Date de visite au format YYYY-MM-DD
nom_prenom string Oui Nom du plaignant
description string Oui Description du problème observé
Exemple
POST /api/inspection
Content-Type: application/json

{
  "nom_etablissement": "Restaurant ABC",
  "adresse": "123 rue Principale",
  "ville": "Montréal",
  "date_visite": "2024-06-15",
  "nom_prenom": "Jean Dupont",
  "description": "Présence de moisissures."
}
DELETE /api/inspection?nom=Nom&date=YYYY-MM-DD

Supprime une demande d’inspection selon le nom de l’établissement et la date.

Paramètres
Nom Type Requis Description
nom string Oui Nom de l’établissement
date string Oui Date de visite au format YYYY-MM-DD
Exemple
DELETE /api/inspection?nom=Restaurant%20ABC&date=2024-06-15
PUT /api/contrevenant/<nom>

Modifie le nom et le propriétaire d’un contrevenant. Cette route est protégée par Basic Auth.

Authentification requise avec HTTP Basic Auth.
Corps JSON attendu
Champ Type Requis Description
nouveau_nom string Oui Nouveau nom de l’établissement
nouveau_proprietaire string Oui Nouveau propriétaire
Exemple
PUT /api/contrevenant/Ancien%20Nom
Authorization: Basic base64(user:password)
Content-Type: application/json

{
  "nouveau_nom": "Nouveau Nom",
  "nouveau_proprietaire": "Nouveau Propriétaire"
}
DELETE /api/contrevenant/<nom>

Supprime logiquement toutes les contraventions d’un établissement. Cette route est aussi protégée par Basic Auth.

Authentification requise avec HTTP Basic Auth.
Exemple
DELETE /api/contrevenant/Nom%20Etablissement
Authorization: Basic base64(user:password)
